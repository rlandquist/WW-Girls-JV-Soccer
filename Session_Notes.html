What we built

A standalone score card template GirlsJVSoccerScoreCard.html — a single HTML file that runs in any browser with no internet required after download.
The card is 440 × 440 px, split into a large photo panel (58%) on the left and a score/info panel (42%) on the right, downloadable as a high-res PNG at 3× scale.
The Waukesha West paw logo is embedded as base64 with the black background removed, so it renders cleanly on the dark card.
"Waukesha West Girls JV Soccer" runs vertically up the left edge as a navy spine label with white italic text.
A standalone preview card template GirlsJVSoccerPreviewCard.html — full 440×440 layout with no photo panel, both team logos at equal size, centered VS divider, prominent date/time banner, location line, and sport spine along the bottom edge.

WW Home / Away toggle — paw logo follows WW to top or bottom slot automatically.

Next game: calendar date picker (formats as m/d), time field, vs. / at toggle, opponent name.
Team photo upload with drag-to-reposition and zoom slider — photo scales to fill frame, stays clamped.
9 background patterns with live mini-preview swatches: Streaks, Carbon, Grid, Halftone, Pulse, Slash, Diamonds, Scatter, Clean.
Opponent logo quick pick — load a logos folder once, then select from 14 conference teams via dropdown.

Session 4 Updates

Continued from Session 3  ·  April 23, 2026  ·  Unified file + persistence + file sync

Unified file

Single file now contains both cards GirlsJVSoccerCard.html — score card and preview card live in one HTML with a Score / Preview mode toggle. Legacy GirlsJVSoccerScoreCard.html and GirlsJVSoccerPreviewCard.html remain in the project for reference only.
Consolidated PAW asset — dropped the duplicate PAW_SMALL base64 in favor of a single PAW that the browser downscales cleanly for the score card slot (71×92) and uses at natural size for the preview card. Saved ~27 KB.

Manage teams and settings modal

New ⚙ Manage teams and settings link opens a modal containing the Spine text editor, a sortable team list (name + filename + ✕), a "+ Add team" button, Clear-all-teams, and the file-sync bar (below).
Auto-fill filename from name — typing "Arrowhead" populates "Arrowhead.png". A dataset.touched flag disables auto-fill once the filename has been manually edited.
Empty-state hint — when no teams exist, the list area shows "No teams yet. Click + Add team to start..." instead of an empty container.
Backdrop click + ESC key close the modal with the same commit semantics as Done (filter blanks → sort alphabetically → save locally → write to linked file).
Edits persist on blur — tab or click away from any field and persistManagerState() writes to localStorage and the linked file. Prevents work loss if the browser closes without clicking Done.

teams.json sync

Two-way auto-sync (Chrome/Edge/Brave only) — link to a teams.json file once, then edits auto-save and the file auto-loads on every subsequent open.
File System Access API + IndexedDB — showSaveFilePicker / showOpenFilePicker for the file choice, IDB (ww-teams-handles) to persist the FileSystemFileHandle across sessions. Permission may require a one-click Reconnect per browser session.
Universal fallback — ⬇ Export and ⬆ Import JSON buttons work in every browser, including Firefox and Safari where auto-sync is unavailable.
Sync bar shows 4 states with a color dot: green = Synced with [filename]; yellow = Permission needed (Reconnect button); gray (FSA) = Saved in this browser only (Save to file / Load from file buttons); gray (no FSA) = Auto-sync unavailable.
Auto-save failures are visible — if writing to the linked file fails (permission revoked, file moved, etc.), a warning toast appears in the bottom-right. Changes remain safe in localStorage.

teams.json format

Now an object, not an array — stores teams AND spine text in one file so both travel together: 
{
  "version": 1,
  "spineText": "Waukesha West Sports Team Name",
  "teams": [
    { "name": "Arrowhead", "file": "Arrowhead.png" },
    ...
  ]
}
Backward compatible — a legacy plain-array teams.json still loads (treated as teams-only with default spine text); on the next save it migrates to the new object format.
parseStoredPayload() filters malformed entries — rows without a string name and file are dropped silently. Non-string spineText falls back to default.

Editable spine text

No more hardcoded text — the vertical spine on the score card and the bottom bar on the preview card both read from a single editable spineText value. Editor lives at the top of the Manage teams and settings modal.
Default: "Waukesha West Sports Team Name" — generic so the file can be handed off to another school/sport without editing HTML. Stored the same moment you change it.
Auto-shrink to fit — long text scales from 12 px default down to 7 px floor. Padding of 16 px at each end keeps a clean blue margin regardless of length. fitSpineText() re-runs on every edit and on mode switch so hidden cards get a real measurement when they become visible.

Empty defaults

No hardcoded conference teams — DEFAULT_OPPONENTS = []. A fresh HTML starts with no teams; populate via "+ Add team" or 📂 Load from file.
"↺ Clear all teams" (renamed from Reset to defaults) — wipes the team list and the linked file's team array, but preserves spine text.
14 conference teams that were hardcoded are no longer in the file. First-time setup requires manually adding them or importing a teams.json.

Time fields with AM/PM toggle

Both time inputs split — the Next game time (score card) and Game time (preview card) are now H:MM text input + AM/PM toggle in the same visual style as the VS/AT toggle.
Flexible escape hatch — free text like TBD or Noon passes through as-is without the AM/PM suffix. _composedTime(hhmm, ampm) detects the H:MM pattern via regex.
Hidden backing fields preserved — #next-time and #game-time still exist as <input type="hidden">, so the render code in updateScore() / updatePreview() is unchanged.

Score card pattern — corner dot mask

Pattern no longer shows through the dots — a destination-out radial gradient carves out the upper-right corner of the pattern canvas after the pattern draws, so the corner dots always sit on clean navy. Mirrors the visual strategy of the preview card's angled pattern fade.
Mask parameters — 120 px radius from corner, fully-erased zone 0–66 px (55% stop), fade zone 66–108 px, pattern fully intact beyond 108 px. Sized with a buffer beyond the visible dot area (~60 px) so there's no residual overlap.
Refactor — the eight drawDotsScore(ctx, w) calls inside each pattern case collapsed into a single call after the switch, with the mask applied between.

Nine-item code review fix pass

html2canvas failures no longer freeze the UI — both download() and copyToClipboard() now have .catch() blocks that restore the canvas (size, scale) and the button state.
Download can't fire in parallel — downloadInProgress guard + button disable prevents rapid double-clicks from triggering simultaneous captures.
loadLogosFolder counts errors — a separate failed counter plus r.onerror handlers mean one bad image no longer wedges the status at "loading…". Shows "(N failed)" in the status if any error.
File inputs reset value — loadPhoto, loadLogo, and loadLogosFolder all clear input.value after processing so the same file can be picked again.
Edits persist on blur — team-row name/file inputs and the spine-text input all have onchange (blur) handlers calling persistManagerState().
Backdrop click + ESC close modal — both route through closeTeamManager() for consistent commit semantics.
Empty-state hint shown when OPPONENTS.length === 0.
Auto-save failure toast — tiny toast system (showToast, container in bottom-right) replaces the silent console.warn when auto-save to the linked file fails.
Backdrop click = Done, not discard — documented and enforced consistent behavior.

New standalone tool GirlsJVSoccerSchedule.html — a companion to the card tool that renders a full-season multi-game schedule card (440 px wide, auto height based on game count). Built as a separate file rather than a 4th mode inside the card tool, so each tool stays focused and the 2736-line card file doesn't balloon further.
Shared team library — reads the same ww-soccer-opponents-v1 localStorage key the card tool writes to, so opponents and logos carry across both tools automatically. No duplicate data entry.

Three-zone game layout — games auto-split by date and status into Results (final games), Next Up (first upcoming game, highlighted with left stripe + translucent accent bg), and Upcoming (remaining scheduled games).

Editable spine text — dedicated input in the editor, persisted to schedule.json. Defaults to "Waukesha West Girls JV Soccer". Same rotated-span pattern as the card tool's sport-label (Arial Black 900 italic 12px, 0.22em letter-spacing, rotated -90deg) — identical font size to the other cards.

Card tool bugfix — frame included in PNG export

Root cause — the decorative white border #card-frame-{mode} is a sibling of #card-{mode}-inner inside .card-wrapper, positioned absolutely over the card. html2canvas only captures a node and its descendants, so capturing the inner element left the frame (a sibling) out of the PNG — the frame showed in the editor but not in downloaded/copied images.
Fix — changed download() and copyToClipboard() in the card tool to use activeCardEl().parentNode (which is the .card-wrapper) as the capture target. The wrapper contains both the inner content and the frame overlay. activeCardEl() itself is unchanged (still used for pattern-canvas sizing, where the inner dimensions are correct).
Same fix applied preemptively to the new schedule tool — its downloadCard() and copyToClipboard() capture document.getElementById('card-inner').parentNode for the same reason.

Visual refinement journey (schedule card)

Layout architecture — first attempt used absolute-positioned spine + left-padded content, which was fragile (CSS specificity override silently forced spine into normal flow, landing it at the top of the card instead of stretching full height). Rebuilt with flexbox: #card-inner is a flex row, spine is a 30 px fixed column, .card-main is flex:1 containing content + footer. Spine now always stretches full card height; footer naturally starts after the spine.
Next Up row logo rescue — .glogo styles were scoped to .grow .glogo only, so the Next Up row's .next-row .glogo rendered as a 0×0 invisible element. Fixed by combining selectors for .glogo, .gdate, .gopp, .gres, .gtime so both row types get proper styling.
Paw debugging — initial base64 copy was truncated from 71 700 chars down to 20 282 (with a character corruption at position 1015), causing the browser to show the broken-image icon. Surgical Python patch replaced the PAW with the full verbatim string from the card tool; verified by decoding to a valid PNG.
Header text compaction — went through several iterations: swapped sport/year order (twice), matched sizes (both 22 px), tightened all spacing (top 22 px → 14 px, sp-year gap 8 px → 4 px, etc.), then pulled the record block up into the header's empty space with margin-top: -34 px instead of moving the header text down. Net result: visually dense top area, no wasted white space.
Footer fill-the-space iteration — landed on 17 px Oswald 700, 0.12 em letter-spacing, 8 px side padding after several rounds. Text fills most of the footer width with visible but small margin on each side. Diagnosed one round where increasing padding didn't help because the text was already wider than the padded area.

What was built (final state)

Final-card headline presets cleaned up — Half Time, Preview, and Game Day buttons removed. Final card now shows only Full Time, Live, Final. Halftime and Preview modes still have their own appropriate presets.
Spine hoisted to full card height — .sport-label moved out of .card-photo, now an absolute-positioned direct child of #card-score-inner with top:0; bottom:0; width:30px; z-index:5. Spans the full 440px height — over the photo area AND alongside the new full-width footer.

Footer is fixed-height (88px) — height:88px; box-sizing:border-box; overflow:hidden. Sized so 6 scorers fit cleanly in a 2×3 grid at default 12px font; 7+ scorers triggers shrink-to-fit. (See "Iterative tuning" below for how we got here.)
Footer mode toggle in editor — new .toggle-row with Next Game (default) / Goal Scorers. Editor field subgroup and card-side content block both swap via setFooterMode() using the hidden attribute on #next-mode-content / #scorers-mode-content.
Both footer titles share styling — "NEXT GAME" and "GOAL SCORERS" both render at 16px Oswald bold uppercase with .08em letter-spacing, top-anchored, with the same 2px underline rule below. Colon was removed from "Next Game:" to match.
Roster integration — loadRosterPlayers() reads localStorage['ww-soccer-roster-v1'] directly (the same key the roster tool writes), normalizes {id, number, firstName, lastName}, and sorts by jersey number with blank-jersey players last. Roster is re-read on every picker render, so a roster edit in another tab is picked up next time the picker is touched.
Scorer picker (editor side) — "+ Add scorer" button generates rows. Each row = roster dropdown (sorted, ending with "+ Custom name…") + goals number input (1–9) + ✕ remove. When custom is selected, the row gains a text input next to the select. A small hint line under the picker reports how many roster players were loaded.
Scorer list (card side) — rendered in #card-scorers-list. Format: #9 SMITH · 2 (jersey number prefix dropped for custom scorers and roster players without a jersey #). 1–3 scorers stack vertically; 4+ flips to a 2-column CSS grid. Opponent logo size balance fix

Symptom — opponent logos (e.g. Arrowhead's) appeared visibly larger than the WW paw on the same card.
